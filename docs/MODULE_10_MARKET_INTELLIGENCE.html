Market Intelligence bridges the gap between internal sales data and external market signals. It combines AI-driven news analysis (5 RSS sources → GPT-4o), live price benchmarking (Google Shopping via SerpAPI), and a partner alert engine that ranks which distributors to call based on a category's price movement and their risk profile. All data is cached locally — no repeated API costs.

Keys are read via os.environ.get() at render time. A status bar at the top of the tab shows ✅ / ❌ connection status for each key. The module degrades gracefully — missing keys disable the relevant sub-feature without crashing.

⚠️
Security Note API keys must be set in the .env file (project root) and loaded by the dashboard entrypoint. Never commit keys to version control. The keys are never exposed to the frontend HTML or logged.

Product Category Auto-Population

The module calls ai.ensure_product_lifecycle() to populate df_product_velocity, from which the category list is built (product_name column). If unavailable, it falls back to a hardcoded list: IT Hardware, Networking, Storage, RAM, Printers, UPS.

Caching

The briefing is cached in st.session_state["_news_cache"] with a 4-hour TTL (_NEWS_CACHE_SECONDS = 14400). A "🔄 Refresh Briefing" button forces re-fetch. The cache age is displayed in minutes when valid.

UI Output

Each story renders as a dark card with a left color-coded border (🔴 High / 🟡 Medium / 🟢 Low impact), plus the market mood banner and any supply chain alert detected by GPT.

ℹ️
Cost Management Each refresh costs approximately 1,500–2,000 GPT tokens (~$0.006 at GPT-4o pricing). The 4-hour cache means a team of 10 users incurs at most 6 API calls per day.

Tab 2: Price Tracker

Users enter a product name + category, then click "Search Prices". The module calls SerpAPI's Google Shopping endpoint and stores results in SQLite. Each search adds one row to the price_history table, enabling trend charts over time.

SerpAPI Call Parameters

params = { "engine": "google_shopping", "q": user_query, "api_key": SERPAPI_KEY, "gl": "in", # India region "hl": "en", "num": 10, }

Price Metrics Computed

MetricFormulaDisplayed As
Avg Pricemean(valid prices)₹ formatted
Min Pricemin(valid prices)₹ formatted
Max Pricemax(valid prices)₹ formatted
Price Spread(max − min) / min × 100% (tooltip explains)

GPT-4o Price Summary

After fetching prices, the top 10 results are formatted as text and sent to GPT-4o for a 2–3 sentence summary covering: price range observed, competitiveness assessment, and one procurement recommendation (e.g. "good time to stock up" or "hold off").

30-Day Trend Chart

A Plotly area chart appears after ≥2 searches on the same query. It shows Avg Price as a line with a min–max shaded range. The chart is populated from price_history filtered to the last 30 days for the exact query string.

💡
Cache-First Logic Before calling SerpAPI, the module checks for a cached result within the last 6 hours from SQLite. If found, no API call is made and the cached result is displayed instantly. This prevents duplicate charges for repeated searches.

Past Searches Log

A collapsible expander at the bottom of the tab shows all searches from the last 30 days: query, category, date, and average price.

If price history exists for the selected category:

SQLite Persistence Layer

All price data is stored in market_prices.db at the project root.

Schema: price_history

CREATE TABLE price_history ( id INTEGER PRIMARY KEY AUTOINCREMENT, product_query TEXT, -- exact search string (case-insensitive lookup) product_category TEXT, -- category tag selected by user search_date TEXT, -- YYYY-MM-DD avg_price REAL, min_price REAL, max_price REAL, results_json TEXT, -- full SerpAPI result as JSON string gpt_summary TEXT, -- GPT-4o procurement summary created_at TEXT -- UTC timestamp for cache cutoff )
ℹ️
DB Initialization _init_db() is called on every tab render with CREATE TABLE IF NOT EXISTS, so no manual setup is required. The DB file is auto-created in the project root.

Urgency Score Formula

Each partner in the alerts tab receives an urgency score [0,1] that determines call priority.

urgency_score = ( 0.40 × churn_probability.clip(0, 1) + 0.30 × revenue_drop_pct.clip(0, 100) / 100 + 0.20 × recency_days.clip(0, 90) / 90 + 0.10 × (total_spend / max_spend_in_category) )
Urgency ScorePriority LabelAction
≥ 0.65🔴 CALL TODAYHighest churn risk + category exposure
0.40 – 0.64🟡 CALL THIS WEEKModerate risk; market timing window
< 0.40🟢 MONITORStable partner; bookmark for next cycle
🎯
Why 40/30/20/10 Weights? Churn probability is the strongest leading indicator of lost revenue — hence 40%. Revenue drop and recency provide lagging confirmation (30%/20%). Spend weight (10%) ensures we prioritize high-value partners even when risk signals are moderate.

GPT-4o Integration Points

The module uses GPT at three places, all routed through the _gpt(system, user, model, max_tokens) helper:

_build_news_briefing() → 1,500 tokens
Converts 25 raw headlines into structured JSON with impact level, category, and recommended action per story. Output: top_stories array + mood + supply alert.
Price summary in Price Tracker → 200 tokens
Summarises top 10 Google Shopping results in 2–3 sentences: price range, competitiveness, procurement recommendation. Cached to SQLite alongside price data.
GPT talking points in Partner Alerts → 500 tokens
Generates personalised outreach talking points for the top 5 partners in a given category, referencing the price trend and framing around urgency to order.
💡
Graceful Degradation All three GPT calls are wrapped in try/except. Errors return "[GPT error: ...]" strings that are checked before rendering. The UI skips the GPT section silently if the key is missing.

Caching Strategy

Cache LayerTypeTTLScope
News briefingst.session_state4 hoursPer browser session (user-level)
Price resultsSQLite price_history6 hours per queryGlobal (all users share)
Price history chartSQLite price_historyPersistent30-day rolling window displayed
Partner alerts spend dataai.df_recent_group_spend300s (Streamlit TTL)Inherited from core engine
⚠️
Session vs Global Cache News briefing is session-scoped — different browsers get independent caches. Price db is global — one search in one browser benefits all users for 6 hours. This is intentional: price data doesn't change user-to-user but news analysis may vary.
